# TRAE Demo Wall 项目概要设计 v0.4

## 1. 版本目标
v0.4 在 v0.3 的基础上新增日志体系，覆盖登录/注册行为与后台关键操作，满足审计追踪与问题定位需求。

## 2. v0.4 核心变更
- 新增登录注册日志表 `sys_auth_log`
- 新增统一操作日志表 `sys_operation_log`
- 在认证回调中埋点登录/注册日志
- 在提交、用户、角色、字典、标签、作品管理接口中埋点操作日志
- 保持增量升级，不清空现有业务数据

## 3. 数据库设计增量

### 3.1 登录注册日志表 (`sys_auth_log`)
记录登录与注册事件（来源 Clerk 回调）。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `user_id` | BIGINT | 系统用户ID，可空 |
| `clerk_id` | VARCHAR(255) | Clerk 用户ID |
| `auth_type` | VARCHAR(50) | 认证类型：`sign_in`/`sign_up` |
| `auth_channel` | VARCHAR(50) | 认证渠道（当前为 `clerk`） |
| `auth_status` | VARCHAR(20) | 认证状态（默认 `success`） |
| `ip_address` | VARCHAR(64) | 请求IP |
| `user_agent` | VARCHAR(512) | 客户端UA |
| `metadata` | JSONB | 扩展信息（邮箱、第三方provider等） |
| `created_at` | TIMESTAMPTZ | 创建时间 |

索引：
- `idx_sys_auth_log_user_id`
- `idx_sys_auth_log_clerk_id`
- `idx_sys_auth_log_auth_type`
- `idx_sys_auth_log_created_at`

### 3.2 操作日志表 (`sys_operation_log`)
记录后台关键写操作及结果，支持审计与追踪。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `operator_id` | BIGINT | 操作人ID，可空 |
| `module` | VARCHAR(50) | 业务模块，如 `users`/`works` |
| `action` | VARCHAR(50) | 操作动作，如 `create`/`update`/`delete` |
| `target_type` | VARCHAR(50) | 目标对象类型 |
| `target_id` | VARCHAR(255) | 目标对象ID |
| `success` | BOOLEAN | 是否成功 |
| `error_message` | VARCHAR(500) | 失败时错误信息 |
| `request_method` | VARCHAR(16) | HTTP 方法 |
| `request_path` | VARCHAR(255) | 请求路径 |
| `ip_address` | VARCHAR(64) | 请求IP |
| `user_agent` | VARCHAR(512) | 客户端UA |
| `payload` | JSONB | 操作摘要数据 |
| `created_at` | TIMESTAMPTZ | 创建时间 |

索引：
- `idx_sys_operation_log_operator_id`
- `idx_sys_operation_log_module_action`
- `idx_sys_operation_log_created_at`

## 4. 日志埋点实现说明

### 4.1 登录/注册埋点
- 入口：`GET /api/auth/callback`
- 行为：调用 `getOrSyncUser({ trigger: 'auth_callback', request })`
- 规则：
  - 首次同步用户记为 `sign_up`
  - 已存在用户记为 `sign_in`
- 落库：
  - `sys_auth_log` 写入认证日志
  - `sys_operation_log` 同步写入 `module=auth` 的操作记录

### 4.2 操作日志埋点范围
已覆盖写接口：
- `POST/PUT/DELETE /api/users`
- `POST/PUT/DELETE /api/roles`
- `POST/PUT/DELETE /api/dictionaries`
- `POST/PUT/DELETE /api/tags`
- `POST/PUT/DELETE /api/works/console`
- `POST /api/submit`

每条日志记录包含：操作人、模块、动作、目标对象、请求元信息、执行结果。

## 5. API 摘要（v0.4 相关）

### Auth
- `GET /api/auth/callback`（登录/注册回调并写入认证日志）

### Logging（数据层）
- `sys_auth_log`：认证事件日志
- `sys_operation_log`：统一操作日志

## 6. 升级策略
- v0.4 采用增量 SQL 脚本，仅新增表和索引。
- 不执行 `TRUNCATE`、不删除历史数据。
- 线上执行前建议先在测试环境验证索引和外键。
