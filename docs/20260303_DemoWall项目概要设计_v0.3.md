# TRAE Demo Wall 项目概要设计 v0.3

## 1. 项目背景
TRAE Demo Wall 是一个用于展示和收录全球范围内 TRAE Demo 作品的平台。旨在促进创意交流，扩大作品影响力，并服务于 TRAE Friends 社区活动。

## 2. 功能模块设计

### 2.1 用户系统 (User System)
支持用户注册和登录，这是作品提交和互动的基础。用户可以维护个人简介、头像等信息。

系统通过角色管理来区分权限：
*   普通用户：可以浏览、点赞、提交作品以及管理自己的作品。
*   管理员：负责审核作品、管理所有内容以及配置系统参数（如审核模式）。

### 2.2 作品管理 (Work Management)
用户通过填写详细表单提交 Demo，内容包括：
*   基础信息：名称、一句话简介、封面图、国家、城市、状态（已上线/开发中/概念）、类别。
*   详细介绍：创作故事、核心亮点、使用场景、界面截图（多图）。
*   链接资源：体验链接、代码仓库。
*   作者信息：团队成员、联系方式（电话/邮箱，可选择是否公开）。

作者可以修改或删除未审核或已发布的作品。

作品展示分为列表页和详情页：
*   列表页：支持按国家、城市、类别、标签筛选，按热度或时间排序，并提供关键词搜索。
*   详情页：展示完整信息，支持多图轮播或平铺，并显示作者信息卡片。

### 2.3 互动系统 (Interaction System)
*   点赞：用户可对作品点赞（限制每人每作品一次）。
*   浏览量：统计作品的访问次数。
*   分享：生成分享卡片或链接（功能预留）。

### 2.4 审核系统 (Audit System)
审核流程分为两种路径：
*   通过：提交 -> 待审核 -> 管理员通过 -> 已发布 (可见)。
*   拒绝：提交 -> 待审核 -> 管理员拒绝 -> 拒绝 (不可见，需修改)。

审核模式支持动态配置：
*   活动模式（标签自动过审）：管理员可设置特定的活动标签（如“2026春季黑客松”），并指定有效时间段。在此期间，带有该标签的作品提交后将自动通过审核，无需人工干预。活动结束后，自动过审功能失效，恢复为人工审核。
*   平时模式（人工审核）：默认情况下，所有提交的作品均需管理员人工审核后才能发布。

### 2.5 系统配置 (System Config)
界面支持中英文切换，内容由用户提供，暂不自动翻译。

### 3.2 数据库设计思路说明

本系统数据库设计主要解决了三个核心问题，确保系统既好用又好维护：

**分类管理**：系统数据（如用户、角色、字典）与业务数据（如作品、标签）分开存储，避免混淆。所有系统基础信息放在 `sys_` 开头的表中，而作品相关内容则统一在 `work_` 开头的表中。

**主次分明**：为了保证列表页加载速度，我们将作品信息拆分为轻量级主表和扩展子表。`work_base` 主表仅存储标题、封面等高频展示字段，而故事背景、多张截图、团队成员等详情页内容则放入 `work_detail` 及其他子表中。

**读写分离**：我们将点赞数、浏览量等高频更新的数据独立存储在 `work_statistic` 表中。这种设计能有效防止因频繁点赞导致的数据库锁竞争，确保系统在多人同时操作时依然稳定流畅。

### 3.3 数据表结构

#### 1. 系统字典主表 (`sys_dict`)
用于存储字典类型定义，如“作品类别”、“国家”、“城市”、“荣誉类型”。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | INT | 主键 |
| `dict_code` | VARCHAR(50) | 字典编码 (Unique, e.g., `work_category`, `country`, `city`, `honor_type`) |
| `dict_name` | VARCHAR(50) | 字典名称 |
| `description` | VARCHAR(255)| 描述 |
| `is_system` | BOOLEAN | 是否系统预设 (不可删) |

#### 2. 系统字典项表 (`sys_dict_item`)
存储具体的字典值，支持动态增量添加。

**V0.3 变更**: 
* 移除 `lang` 字段。
* 新增 `label_i18n` (JSONB) 字段，用于存储多语言标签。
* 每一行记录代表一个唯一的业务含义，不再按语言分行存储。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | INT | 主键 |
| `dict_code` | VARCHAR(50) | 关联字典编码 |
| `item_label` | VARCHAR(100)| 默认显示标签 (Fallback) |
| `label_i18n` | JSONB | 多语言标签 (e.g., `{"zh-CN": "中国", "en-US": "China"}`) |
| `item_value` | VARCHAR(100)| 存储值 (e.g., 'CN', 'tool', 'community_pick') |
| `sort_order` | INT | 排序权重 |
| `status` | BOOLEAN | 启用状态 (1=启用, 0=禁用) |

#### 3. 系统用户表 (`sys_user`)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `username` | VARCHAR | 用户名 |
| `email` | VARCHAR | 邮箱 (登录账号) |
| `phone` | VARCHAR | 手机号 (选填) |
| `password_hash` | VARCHAR | 密码哈希 |
| `avatar_url` | VARCHAR | 头像链接 |
| `bio` | TEXT | 个人简介 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

#### 4. 系统角色表 (`sys_role`)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | INT | 主键 |
| `role_code` | VARCHAR | 角色编码 (e.g., `admin`, `user`) |
| `role_name` | VARCHAR | 角色名称 (e.g., `管理员`, `普通用户`) |
| `description` | VARCHAR | 描述 |

#### 5. 用户角色关联表 (`sys_user_role`)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `user_id` | BIGINT | 用户ID |
| `role_id` | INT | 角色ID |

#### 6. 作品基础表 (`work_base`)
仅包含作品核心元数据，去除大字段、联系信息和状态字段，保持轻量。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `user_id` | BIGINT | 关联用户ID (Owner) |
| `title` | VARCHAR | 作品名称 |
| `summary` | VARCHAR(255)| 一句话简介 |
| `cover_url` | VARCHAR | 封面图链接 (列表页展示用) |
| `country_code` | VARCHAR | 国家 (关联 `sys_dict_item.item_value`) |
| `city_code` | VARCHAR | 城市 (关联 `sys_dict_item.item_value`) |
| `category_code` | VARCHAR | 类别 (关联 `sys_dict_item.item_value`) |
| `dev_status_code`| VARCHAR | 开发状态 (关联 `sys_dict_item.item_value`) |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

#### 7. 作品详情表 (`work_detail`)
存储大文本和非检索字段，垂直分表优化性能。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `work_id` | BIGINT | 主键 (关联 work_base.id) |
| `story` | TEXT | 创作故事 |
| `highlights` | JSON | 核心亮点 (数组) |
| `scenarios` | JSON | 使用场景 (数组) |
| `demo_url` | VARCHAR | 体验链接 |
| `repo_url` | VARCHAR | 代码仓库链接 |

#### 8. 作品图片表 (`work_image`)
独立管理作品图片（截图/详情图），支持排序。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `work_id` | BIGINT | 关联作品ID |
| `image_url` | VARCHAR | 图片链接 |
| `image_type` | VARCHAR | 图片类型 (e.g., `screenshot`, `detail`) |
| `sort_order` | INT | 排序权重 |
| `created_at` | DATETIME | 创建时间 |

#### 9. 团队信息表 (`work_team`)
存储作品的团队成员列表及团队介绍。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `work_id` | BIGINT | 关联作品ID |
| `team_intro` | TEXT | 团队介绍 (文本) |
| `members` | TEXT | 成员名单 (文本，顿号分隔) |
| `contact_phone` | VARCHAR | 联系电话 (可选) |
| `contact_email` | VARCHAR | 联系邮箱 (可选) |

#### 10. 作品状态与统计表 (`work_statistic`)
解耦状态管理，分离审核状态与发布状态，包含浏览量统计。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `work_id` | BIGINT | 主键 (关联 work_base.id) |
| `audit_status` | INT | 审核状态: 0=待审, 1=通过, 2=拒绝 |
| `display_status` | INT | 展示状态: 0=下架, 1=上架 (审核通过且用户开启) |
| `view_count` | BIGINT | 浏览量 (总计) |
| `like_count` | BIGINT | 点赞量 (总计) |
| `last_audit_at` | DATETIME | 最后审核时间 |

#### 11. 作品荣誉/推荐表 (`work_honor`)
用于给作品打上“社区推荐”、“城市之星”等官方荣誉标签。

**V0.2 变更**: 改为直接关联 `sys_dict_item` 的 `id`，移除 `honor_code` 和 `honor_name` 字段，以解决多语言环境下的唯一性问题。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `work_id` | BIGINT | 关联作品ID |
| `honor_item_id` | BIGINT | 荣誉字典项ID (关联 `sys_dict_item.id`) |
| `granted_at` | DATETIME | 授予时间 |
| `granted_by` | BIGINT | 授予人 (管理员ID) |

#### 12. 审核记录表 (`work_audit_log`)
记录完整的审核历史。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `work_id` | BIGINT | 关联作品ID |
| `auditor_id` | BIGINT | 审核员ID |
| `prev_status` | INT | 审核前状态 (0=待审核, 1=已通过, 2=已拒绝) |
| `new_status` | INT | 审核后状态 (0=待审核, 1=已通过, 2=已拒绝) |
| `reason` | VARCHAR | 审核意见/拒绝理由 |
| `created_at` | DATETIME | 审核时间 |

#### 13. 作品标签表 (`work_tag`)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | INT | 主键 |
| `name` | VARCHAR | 标签名 (唯一) |
| `is_auto_audit` | BOOLEAN | 是否开启自动过审 (默认 false) |
| `audit_start_time` | DATETIME | 自动过审开始时间 (可选) |
| `audit_end_time` | DATETIME | 自动过审结束时间 (可选) |

#### 14. 作品-标签关联表 (`work_tag_relation`)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `work_id` | BIGINT | 作品ID |
| `tag_id` | INT | 标签ID |

#### 15. 作品点赞记录表 (`work_like`)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `user_id` | BIGINT | 用户ID |
| `work_id` | BIGINT | 作品ID |
| `created_at` | DATETIME | 点赞时间 |


## 4. 接口概要 (API Summary)

### Auth
*   `POST /api/auth/register`
*   `POST /api/auth/login`
*   `GET /api/auth/me`
*   `PUT /api/auth/profile` (更新个人信息)

### Dictionaries & Tags
*   `GET /api/dicts/:code` (获取字典项，如 country, city, category)
*   `GET /api/tags` (获取热门/所有标签)

### Works
*   `GET /api/works` (列表，支持 query params: `page`, `limit`, `sort`, `country`, `city`, `category`, `tag`)
*   `GET /api/works/:id` (详情，包含团队、图片、标签等)
*   `POST /api/works` (创建，需登录)
*   `PUT /api/works/:id` (更新，需作者或管理员)
*   `DELETE /api/works/:id` (删除)
*   `POST /api/works/:id/like` (点赞)
*   `POST /api/works/:id/view` (增加浏览量)

### Upload
*   `POST /api/upload` (上传图片)

### Admin
*   `GET /api/admin/works` (获取待审核/所有作品)
*   `POST /api/admin/works/:id/audit` (审核通过/拒绝)
*   `POST /api/admin/works/:id/honor` (授予荣誉)
*   `GET /api/admin/tags` (获取标签管理列表)
*   `PUT /api/admin/tags/:id` (配置标签自动过审规则)
*   `DELETE /api/admin/tags/:id` (删除违规标签)
