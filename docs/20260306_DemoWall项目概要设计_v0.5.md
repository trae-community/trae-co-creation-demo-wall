# TRAE Demo Wall项目概要设计 v0.5

## 1. 版本目标
v0.5 在 v0.4 的基础上实现字典项层级关系支持，解决国家与城市等具有层级关系的数据管理需求，提升筛选体验。

## 2. v0.5 核心变更
- 新增 `sys_dict_item.parent_value` 字段，支持字典项层级关联
- 实现提交/编辑作品时的城市层级过滤（选择国家后只显示对应城市）
- 后台字典管理支持设置和编辑 `parentValue` 字段
- API 接口支持 `parentValue` 的创建和更新
- 保持向后兼容，不影响现有无层级关系的字典

## 3. 数据库设计增量

### 3.1 字典项表扩展 (`sys_dict_item`)

**新增字段**:

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `parent_value` | VARCHAR(100) | 父级值，用于建立层级关系（如城市→国家） |

**索引**:
- `idx_sys_dict_item_parent_value`：优化基于父级值的查询性能

**示例数据**:
```sql
-- 国家数据（无需 parent_value）
INSERT INTO sys_dict_item (dict_code, item_label, item_value, label_i18n) 
VALUES ('country', '中国', 'CN', '{"zh-CN": "中国", "en-US": "China"}');

-- 城市数据（需要 parent_value）
INSERT INTO sys_dict_item (dict_code, item_label, item_value, label_i18n, parent_value) 
VALUES ('city', '北京', '110100', '{"zh-CN": "北京", "en-US": "Beijing"}', 'CN');
```

## 4. 前端实现说明

### 4.1 层级过滤逻辑

**适用页面**:
- `/[language]/submit` - 提交作品表单
- `/[language]/works/edit/[id]` - 编辑作品表单

**核心实现**:
```typescript
const filteredCities = useMemo(() => {
  if (!selectedCountry) return [];
  return availableCities.filter(city => city.parentValue === selectedCountry);
}, [availableCities, selectedCountry]);
```

**交互规则**:
- 未选择国家时，城市下拉框禁用
- 切换国家时，自动清空已选城市并重新加载可选城市列表
- 显示格式：`显示 1-3 共 5 条记录`（分页场景）

### 4.2 后台管理功能

**字典管理页面** (`/[language]/console/dictionaries`):

- 编辑字典项时，若 `dictCode === 'city'`，显示"所属国家"输入框
- `parentValue` 为可选字段，仅对需要层级的字典项设置
- 支持输入任意国家代码（如 CN, US, JP）

## 5. API 设计（v0.5 相关）

## 6. 数据结构设计

### 6.1 TypeScript 类型定义

**DictionaryItem** (`src/lib/types.ts`):
```typescript
export interface DictionaryItem {
  id: string;
  dictCode: string;
  itemLabel: string;
  itemValue: string;
  parentValue?: string; // v0.5 新增：父级值
  labelI18n?: Record<string, string>;
  sortOrder: number;
  status: boolean;
}
```

**DictItem** (`src/app/[language]/console/dictionaries/page.tsx`):
```typescript
interface DictItem {
  id: string;
  dictCode: string;
  itemLabel: string;
  itemValue: string;
  labelI18n?: Record<string, string> | null;
  parentValue?: string | null; // v0.5 新增
  sortOrder: number;
  status: boolean;
}
```

### 6.2 Prisma Schema

**SysDictItem** (`prisma/schema.prisma`):
```prisma
model SysDictItem {
  id          BigInt   @id @default(autoincrement())
  dictCode    String   @map("dict_code")
  itemLabel   String   @map("item_label")
  labelI18n   Json?    @map("label_i18n")
  itemValue   String   @map("item_value")
  parentValue String?  @map("parent_value") // v0.5 新增
  sortOrder   Int?     @default(0) @map("sort_order")
  status      Boolean? @default(true)
  
  dict SysDict @relation(fields: [dictCode], references: [dictCode], onDelete: Cascade)
  workHonors WorkHonor[]
  
  @@map("sys_dict_item")
}
```

## 7. 应用场景与扩展

### 7.1 当前应用
- **国家 → 城市**：选择国家后过滤城市列表
- 适用字典：`country`（父级）、`city`（子级）

### 7.2 未来扩展
同样的模式可应用于其他层级场景：

**省市区三级联动**:
```sql
-- 省级
INSERT INTO sys_dict_item (dict_code, item_label, item_value, parent_value) 
VALUES ('province', '广东省', 'GD', NULL);

-- 市级
INSERT INTO sys_dict_item (dict_code, item_label, item_value, parent_value) 
VALUES ('city', '广州市', '440100', 'GD');

-- 区级
INSERT INTO sys_dict_item (dict_code, item_label, item_value, parent_value) 
VALUES ('district', '天河区', '440106', '440100');
```

**分类层级**:
- 一级分类 → 二级分类 → 三级分类
- 通过 `parentValue` 建立树形结构

**组织架构**:
- 部门 → 子部门 → 团队

## 8. 向后兼容性

### 8.1 数据兼容
- `parentValue` 为可选字段，已有字典项无需强制设置
- 未设置 `parentValue` 的城市在所有国家下均显示（降级兼容）

### 8.2 接口兼容
- 现有 API 响应增加 `parentValue` 字段（可为 `null`）
- 前端组件根据是否有 `parentValue` 决定是否显示过滤逻辑

### 8.3 前端兼容
- 非城市字典不受影响（不显示"所属国家"输入框）
- 已有表单逻辑保持不变

## 11. 性能优化

### 11.1 数据库索引
- `idx_sys_dict_item_parent_value`：加速基于父级值的查询

### 11.2 前端优化
- 使用 `useMemo` 缓存过滤结果，避免重复计算
- 城市列表数据量小（通常 < 100），内存过滤性能足够

## 12. 总结

v0.5 通过简单的 `parentValue` 字段实现了灵活的字典层级关系，核心优势：

1. **轻量级改造**：仅需添加一个可选字段
2. **向后兼容**：不影响现有数据和功能
3. **高扩展性**：可复用于多种层级场景
4. **用户体验提升**：层级过滤减少误操作，提升填写效率

该设计为平台未来的地域筛选、分类管理等高级功能奠定了基础。
